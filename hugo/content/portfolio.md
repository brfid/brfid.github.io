## API Reference and Developer Docs

**[Threat Feeds OpenAPI Specification](https://docs.domaintools.com/feeds/openapi/)** (2025)
OpenAPI 3.0.3 specification for DomainTools Feeds — real-time and batch endpoints, download
workflow, reusable components, examples, and error contracts. Built for code generation,
Swagger UI rendering, and contract validation.

**[DomainTools API Reference](https://docs.domaintools.com/api/)** (2025)
API reference documentation for DomainTools data products and endpoints, including the
[Threat Intelligence Feeds API](https://docs.domaintools.com/api/threat-feeds/).

**[DomainTools Python SDK](https://docs.domaintools.com/python-sdk/)** (2025)
Developer guide and API reference: installation, authentication, usage patterns, method reference.

**[DomainTools Product Documentation](https://docs.domaintools.com)** (2025)
User-facing documentation for DomainTools security intelligence products and workflows.

## Research and Commissioned History

**[The Creation and Administration of Unique Identifiers, 1967–2017](https://www.icann.org/en/system/files/files/creation-administration-unique-identifiers-1967-2017-18nov20-en.pdf)** (2020)
Comprehensive history of internet unique identifier creation and administration from ARPANET
origins through 2017. Co-authored with Russ Mundy; jointly commissioned by ICANN and Google.
Foreword by Vint Cerf and Steve Crocker.

**[Work-From-Home and COVID-19: Trajectories of Endpoint Security Management in a Security Operations Center](https://www.usenix.org/conference/usenixsecurity23/presentation/jones)** (2023)
Peer-reviewed paper at USENIX Security '23. Co-authored historical analysis tracing endpoint
security management from the 1970s through the COVID-19 WFH transition, complementing a
1,000-hour SOC ethnography study.

**[Edge Cryptography and the Codevelopment of Computer Networks and Cybersecurity](https://doi.org/10.1109/MAHC.2016.49)** (2016)
Peer-reviewed article in IEEE Annals of the History of Computing.
